## The token system

<p className="text-lg text-muted-foreground leading-[1.6] m-0 max-w-[680px]">
  Instead of scattering pixel values across components, every gap in the

  consistent, purpose-built spacing scale that can be tuned from a single
  place and applied across every product at once.
</p>

<div className="rounded-xl border-2 border-brand-primary-200 dark:border-brand-primary-800 bg-brand-primary-50 dark:bg-brand-primary-950 p-6" style={{ marginTop: '1.5rem' }}>

  <p className="text-base text-brand-primary-700 dark:text-brand-primary-300 leading-[1.6] m-0">

  </p>

## The density-aware system

PROCERTUS is a **multi-density system**. The same component library serves two
fundamentally different audiences, and each audience needs a different relationship
with space. This is not a screen-size concern. It is a product-level decision:
density is driven by **who** is using the interface and **why**.

      density uses tight spacing to keep more data visible without scrolling:
      compact cards, dense tables, narrow margins. Speed and efficiency are the
      priority.

## The accordion system

<p className="text-lg text-muted-foreground leading-[1.6] m-0 max-w-[680px]">
  Procertus spacing is responsive. Tokens automatically shift at the 768px
  breakpoint so the interface adapts to the device without manual overrides.
</p>

<div className="grid grid-cols-1 md:grid-cols-2 gap-6" style={{ marginTop: '1.5rem' }}>
  <div className="rounded-xl border border-border bg-card p-6 flex flex-col">
    <span className="font-semibold text-foreground block" style={{ fontSize: '1.5rem' }}>Stable glue</span>
    <span className="uppercase block mb-3" style={{ fontSize: '12px', fontWeight: 600, letterSpacing: '0.1em', color: 'var(--muted-foreground)' }}>Micro and element</span>
    <p className="text-sm text-muted-foreground m-0" style={{ lineHeight: 1.6 }}>
      The internal "glue" that holds icons to labels and pads containers stays
      stable or grows slightly on mobile to accommodate touch targets. These
      values ensure tap areas remain comfortable without distorting the component's
      internal rhythm.
    </p>
  </div>
  <div className="rounded-xl border border-border bg-card p-6 flex flex-col">
    <span className="font-semibold text-foreground block" style={{ fontSize: '1.5rem' }}>Structural air</span>
    <span className="uppercase block mb-3" style={{ fontSize: '12px', fontWeight: 600, letterSpacing: '0.1em', color: 'var(--muted-foreground)' }}>Region and boundary</span>
    <p className="text-sm text-muted-foreground m-0" style={{ lineHeight: 1.6 }}>
      The large-scale structural spacing compresses on mobile where viewport
      real estate is scarce, then expands on desktop where the screen can
      afford to breathe. This is where the accordion effect is most visible:
      Region can jump from 20px to 32px (or 24px to 48px in Spacious mode).
    </p>
  </div>
</div>

<p className="text-sm text-muted-foreground m-0 max-w-[680px]" style={{ marginTop: '1.5rem', lineHeight: 1.6 }}>
  The system is built to scale consistently. If we adjust a token's value
  in the future, the change propagates globally to maintain visual harmony
  across every component, page, and density context.
</p>
