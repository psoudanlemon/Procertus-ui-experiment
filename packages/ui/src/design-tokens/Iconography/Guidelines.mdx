## 02. The stroke and its scale

<p className="text-base text-muted-foreground leading-[1.6] m-0 max-w-[680px]">
  Every icon renders exclusively in the <strong>Stroke</strong> variant at a baseline weight of
  <strong> 1.33px</strong>. We chose this weight because it matches the sophistication of the
  PROCERTUS brand. This precision must also hold at every size.
</p>

<div className="mt-6 flex flex-col gap-4">
  <div className="grid grid-cols-1 md:grid-cols-2 gap-4">
    <div className="rounded-xl border border-border bg-card p-6">
      <h4 className="font-sans text-base font-bold text-foreground m-0 mb-3">The 1.33px baseline</h4>
      <p className="text-sm text-muted-foreground leading-[1.6] m-0">
        A heavier stroke feels consumer-grade: rounded, friendly, informal. A thinner stroke
        feels fragile: elegant but uncertain. At 1.33px, symbols carry exactly enough visual
        weight to stand beside dense data tables and legal-weight text without either disappearing
        or shouting. The weight says "this system is exact."
      </p>
    </div>
    <div className="rounded-xl border border-border bg-card p-6">
      <h4 className="font-sans text-base font-bold text-foreground m-0 mb-3">Proportional scaling</h4>
      <p className="text-sm text-muted-foreground leading-[1.6] m-0">
        A 1.33px stroke looks sharp at 16px, but at 48px it would appear paper-thin.
        To preserve visual density across all sizes, we generated a custom curve that gently
        increases the stroke weight as icons scale up. Whether an icon sits in a sidebar or
        anchors an empty state, it always carries the same engineered authority.
      </p>
    </div>
  </div>
  <a href="?path=/story/design-tokens-iconography--proportional-scale" className="font-semibold text-brand-primary-700 underline underline-offset-4 hover:text-brand-primary-900">View the proportional scale</a>

## 03. Branded vs. neutral

<p className="text-base text-muted-foreground leading-[1.6] m-0 max-w-[680px]">
  Icons in the application shell carry brand color to scale our identity into the navigation
  layer. Everywhere else, icons stay neutral. This split is intentional: the sidebar is the
  one surface that is always visible, always orienting, so it earns the branded treatment.
  The rest of the interface stays quiet so the content can speak.
</p>

<div className="mt-6 flex flex-col gap-4">
  <div className="grid grid-cols-1 md:grid-cols-2 gap-4">
    <div className="rounded-xl border border-border bg-card p-6">

      <p className="text-sm text-muted-foreground m-0" style={{ lineHeight: 1.6 }}>
        Navigation icons in the sidebar carry a branded color that is always on, providing a
        visible thread of identity through every screen. The user never has to search for where
        they are: the colored symbols tell them. By confining brand color to this single surface,
        we avoid diluting it across the entire UI.
      </p>
    </div>
    <div className="rounded-xl border border-border bg-card p-6">
      <span className="font-semibold text-foreground block" style={{ fontSize: '1.5rem' }}>Everything else</span>

      <p className="text-sm text-muted-foreground m-0" style={{ lineHeight: 1.6 }}>
