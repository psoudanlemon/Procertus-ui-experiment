    All Hugeicons in the PROCERTUS system render at a <strong>1.33px</strong> stroke weight.
    This value is not arbitrary: it provides the optimal balance between precision and sturdiness
    at our standard icon sizes (16px and 20px). Thinner strokes disappear at small sizes, heavier
    strokes compete with text. The 1.33px weight ensures icons remain crisp on both standard
    and high-density displays while maintaining visual parity with our typography.

## 02. Navigational symbolism

<p className="text-base text-muted-foreground leading-[1.6] m-0 max-w-[680px]">
  For our primary navigation (the sidebar), we use the Hugeicons <strong>Two-Tone</strong> variant
  to create visual depth and extend our brand's checkmark logic into the user's path. The dual-color
  treatment gives navigation icons a sense of dimensionality that single-color icons cannot achieve,
  subtly reinforcing the PROCERTUS identity at every level of the interface.

  The two-tone color mapping follows a strict hierarchy: the primary color dominates the icon's
  structure while the accent remains subtle, providing depth without competing for attention.
</p>

## 03. Actionable clarity

<p className="text-base text-muted-foreground leading-[1.6] m-0 max-w-[680px]">
  For functional actions (buttons, inputs, inline commands), we use the high-legibility Hugeicons
  <strong> Stroke</strong> variant. Where Two-Tone creates depth for orientation, Stroke creates
  focus for execution. The single-color treatment ensures maximum contrast and instant recognition
  at the moment a user commits to an action.
</p>

### Color and behavior

<div className="grid grid-cols-1 md:grid-cols-2 gap-6 mt-6">
  <div className="rounded-xl border border-border bg-card p-6">
    <span className="font-semibold text-foreground block" style={{ fontSize: '1.5rem' }}>Static</span>
    <span className="uppercase block mb-3" style={{ fontSize: '12px', fontWeight: 600, letterSpacing: '0.1em', color: 'var(--muted-foreground)' }}>One-color, foreground</span>
    <p className="text-sm text-muted-foreground m-0" style={{ lineHeight: 1.6 }}>
      Stroke icons in buttons and inputs use the component's standard foreground color.
      They inherit <code className="text-sm bg-muted px-1.5 py-0.5 rounded">currentColor</code> from
      their parent, ensuring they always match the text they accompany without any manual color assignment.
    </p>
  </div>
  <div className="rounded-xl border-2 border-brand-accent-200 dark:border-brand-accent-800 bg-brand-accent-50 dark:bg-brand-accent-950 p-6">
    <span className="font-semibold text-foreground block" style={{ fontSize: '1.5rem' }}>Hover transition</span>
    <span className="uppercase block mb-3" style={{ fontSize: '12px', fontWeight: 600, letterSpacing: '0.1em', color: 'var(--muted-foreground)' }}>The dialect shift</span>
    <p className="text-sm text-muted-foreground m-0" style={{ lineHeight: 1.6 }}>
      When a navigational Two-Tone icon is hovered, it transitions to the Stroke variant
      and adopts the sidebar accent foreground color. This "dialect shift" from orientation
      to action creates a clear visual signal that the user is about to act, bridging
      the gap between <em>where I am</em> and <em>what I can do</em>.
    </p>
  </div>
</div>

## 04. The proximity rule

<div className="rounded-xl border border-border bg-card p-6 mt-6">
  <p className="text-base text-muted-foreground leading-[1.6] m-0 max-w-[680px]">
    Icons are always paired with text using the <strong>Micro</strong> spacing token (<code className="text-sm bg-muted px-1.5 py-0.5 rounded">4px</code>).
    This tight coupling creates a single, unified <strong>Symbolic Action</strong> grouping: the icon and its
    label read as one atomic element rather than two adjacent objects. This aligns with the proximity
    principle documented in our <Button variant="link" className="p-0 h-auto text-base inline" asChild><a href="?path=/docs/design-tokens-spacing-guidelines--docs">Spacing Guidelines</a></Button>,
    where the Micro token is reserved for binding elements that share a single semantic function.
  </p>
</div>
