## 02. Background application

Each color treatment is tied to specific background conditions.
The mode is determined by the surface, not by preference: color on light,
white on dark, black on monochrome print stock. The background rules story
maps every approved, conditional, and forbidden combination.

## 04. Usage and sizing

Each variant has specific minimum reproduction sizes for digital and print
contexts. Below these thresholds the logotype fails fidelity checks and
should not be used at all.

<div className="flex flex-col gap-6 mt-6">

  <div className="grid grid-cols-1 md:grid-cols-2 gap-6">
    <div className="rounded-xl border border-border bg-card p-6">
      <h4 className="font-sans text-base font-bold text-foreground m-0 mb-3">Horizontal variants</h4>
      <p className="text-sm text-muted-foreground leading-[1.6] m-0">
        The horizontal logotype (with or without tagline) requires a minimum
        width of <strong>320px</strong> in digital contexts and <strong>38mm</strong> in print.
        Use the tagline version when introducing the brand for the first time.
        Drop the tagline in repeated placements like navigation headers.
      </p>
    </div>
    <div className="rounded-xl border border-border bg-card p-6">
      <h4 className="font-sans text-base font-bold text-foreground m-0 mb-3">Vertical variants</h4>
      <p className="text-sm text-muted-foreground leading-[1.6] m-0">
        The vertical logotype (with or without tagline) requires a minimum
        width of <strong>200px</strong> in digital contexts and <strong>38mm</strong> in print.
        Preferred for centered compositions, certificates, and layouts where
        vertical emphasis works better than horizontal.
      </p>
    </div>
  </div>

  <div className="grid grid-cols-1 md:grid-cols-2 gap-6">
    <div className="rounded-xl border border-border bg-card p-6">
      <h4 className="font-sans text-base font-bold text-foreground m-0 mb-3">Logomark</h4>
      <p className="text-sm text-muted-foreground leading-[1.6] m-0">
        The standalone pictogram requires a minimum size of <strong>44px</strong> for
        digital and <strong>38mm</strong> for print. Below these thresholds, do not
        use the pictogram at all. It may also appear as a low-opacity watermark
        (below 10%) anchored to the bottom-right of branded surfaces like
        certificates and status pages.
      </p>
    </div>
    <div className="rounded-xl border border-border bg-card p-6">
      <h4 className="font-sans text-base font-bold text-foreground m-0 mb-3">Digital vs. print</h4>
      <p className="text-sm text-muted-foreground leading-[1.6] m-0">
        Digital contexts use pixel-based minimums and CSS filters for color
        treatment (<code className="text-xs bg-muted px-1 py-0.5 rounded">brightness-0 invert</code> for white, <code className="text-xs bg-muted px-1 py-0.5 rounded">grayscale brightness-0</code> for black).
        Print contexts use millimetre-based minimums. Never manually create
        white or monochrome SVG variants.
      </p>
    </div>
  </div>

</div>

## 05. Light and dark mode

In digital interfaces the logotype must adapt to the active color mode.
The color variant is used in light mode, the white variant in dark mode.
This switch is handled entirely through CSS filters applied to the same
SVG source file: <code className="text-xs bg-muted px-1 py-0.5 rounded">brightness-0 invert</code> produces the white
version on dark surfaces. There is no separate white SVG.
